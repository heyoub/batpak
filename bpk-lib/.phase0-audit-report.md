## Findings beyond planning snapshot

### F1. Real compilation failure surfaced — FIXED (drive-by)

The planning-time audit ran `cargo clippy --workspace --no-deps` without
`--all-targets`, which skipped test target compilation. Running with
`--all-targets` revealed:

```
error[E0063]: missing field `status` in initializer of `harness_lints::LedgerEntry`
   --> tools/integrity/src/harness_lints.rs:796:9
```

This is the canonical signature of a half-merged change: the struct
`LedgerEntry` (defined at line 236) was extended with a
`status: Option<String>` field in a recent commit, but the in-module
test fixture `complete_entry(...)` at line 795 was not updated. Both
synthetic tests
(`synthetic_well_formed_ledger_entry_is_accepted`,
`synthetic_malformed_ledger_entry_is_rejected`) were failing to
compile.

**Action taken:** added `status: Some("green".to_owned())` to the
fixture initializer (`VALID_STATUS` contains `["green", "amber", "red",
"unmeasured"]`; `green` is the only sensible default for a fixture
labeled `synthetic_well_formed`). Verified with
`cargo test -p batpak-integrity synthetic`: 2 passed, 0 failed.

This is within the audit's fix-scope per the plan: "unambiguously
orphaned ... from a half-merged change". Will be committed in a
separate, clearly-labeled commit before slice work.
